# Life Manager 落地改造计划

## 1. 当前底座

当前项目包含：

- `admin-web`：Vue 3 + TypeScript + Vite + Soybean Admin + Naive UI。
- `admin-service`：RuoYi-Vue-Plus 5.6.1，Spring Boot 3.5，Sa-Token，MyBatis-Plus。
- 已有能力：登录、菜单、权限、用户、角色、字典、参数、日志、OSS、SSE、代码生成、Docker 部署脚本。

虽然最初设想是 React，但当前 Vue 底座已经覆盖了大量后台基础能力。第一版建议接受 Vue，优先把系统做出来。

## 2. 改造原则

1. 保留底座能力，不重写登录、权限、菜单、OSS、SSE。
2. 删除或隐藏无关演示页面，避免后台味太重。
3. Life Manager 业务模块独立放置，不直接混进系统管理模块。
4. 先做项目视角，再补完整功能视角。
5. 首页优先改成产品型仪表盘，而不是传统数据看板。
6. 敏感信息第一版就按安全要求处理，不先明文凑合。

## 3. 保留能力

后端保留：

- 登录认证
- 用户体系
- 菜单权限
- 角色权限
- 字典配置
- 参数配置
- 操作日志
- 登录日志
- OSS 文件能力
- SSE 推送能力
- Docker 部署基础
- 代码生成器

前端保留：

- 基础布局
- 登录页
- 菜单渲染
- 权限路由
- Naive UI 组件体系
- 主题能力
- 表格、表单、上传等公共组件
- SSE 客户端工具
- OSS 上传组件

## 4. 需要弱化或隐藏的内容

第一阶段可以隐藏：

- 租户管理
- 租户套餐
- 部门管理
- 岗位管理
- 工作流
- 监控页面
- 缓存页面
- 在线用户
- 代码生成入口
- Demo 页面
- 关于页面

这些能力不是删除，只是不作为 Life Manager 的主界面出现。后续需要时可以在设置或开发者区域恢复。

## 5. 业务模块规划

第一批主导航建议改成：

- 首页
- 项目
- 待确认
- 设置

日程、清单、记录、资产、图册、AI 助手这类内容不再默认平铺为主导航。它们是功能块或能力，应该根据项目类型和项目方案进入项目内部 tab、概览摘要、时间轴或项目管理页配置。系统管理相关页面可以放到“设置”或“系统设置”里。

## 6. 第一阶段目标

第一阶段目标是把底座从“若依后台”变成“Life Manager 产品雏形”。

需要完成：

1. 系统品牌替换为 Life Manager。
2. 首页替换为 Life Manager 首页。
3. 新增项目管理页面。
4. 新增项目内部首页页面。
5. 新增项目管理配置页面。
6. 菜单调整为 Life Manager 的信息架构。
7. 保留系统管理能力，但弱化入口。

第一阶段可以先使用静态假数据，不急着接数据库。

## 7. 第一阶段页面

### 7.1 系统首页

首页需要展示：

- 今日概览
- 今日待办
- 即将到期
- 项目总览
- AI 今日建议
- 本月统计
- 快捷入口
- 实时状态

首页重点是“每天打开能知道该关注什么”。

### 7.2 项目管理

项目管理页需要展示：

- 全部项目
- 游戏项目
- 旅行项目
- 学习项目
- 生活项目
- 归档项目

项目卡片需要体现：

- 项目类型
- 项目状态
- 最近动态
- 下一个提醒
- 关联内容摘要
- AI 整理状态

### 7.3 无限暖暖项目内部首页

页面需要展示：

- 概览摘要：当前版本、今日目标、本周目标、即将结束活动、素材收集、图册记录、资产风险、最近时间轴
- 任务 tab：日常、周常、活动目标、自定义目标、历史记录
- 活动与版本 tab：当前版本、活动列表、提醒、待归档、版本归档
- 图册 tab：照片、图册、版本/活动/目标关联
- 时间轴 tab：按天、按周、按版本回顾长期记录
- 账号资产 tab：账号、平台、支付凭证、兑换码状态
- AI 建议：放在概览和时间轴，不作为独立一级 tab

### 7.4 无限暖暖项目管理

页面需要展示：

- 基础信息
- 功能块实例配置
- 概览摘要规则
- 任务重置规则
- 活动提醒与归档规则
- 字段配置
- 时间轴写入规则
- 账号资产、图册同步和 AI 自动化设置
- 外部通知

### 7.5 日本旅行项目内部首页

页面需要展示：

- 出发倒计时
- 近期行程
- 行前清单
- 机票酒店资料
- 保险和证件资料
- 预算概览
- 天气与地点
- 图册预览
- 旅行笔记
- AI 行前建议

### 7.6 日本旅行项目管理

页面需要展示：

- 基础信息
- 行程规划
- 同行人员
- 清单模板
- 资产资料
- 图册同步
- 提醒规则
- AI 设置

## 8. 第二阶段目标

第二阶段开始接后端业务。

优先顺序：

1. 项目模块
2. 项目类型、方案和功能块实例配置
3. 任务目标、版本活动和时间轴模块
4. 素材、资产、图册模块
5. 概览摘要规则和待确认收件箱
6. AI 摘要模块
7. 外部通知模块

第二阶段完成后，页面不再依赖静态假数据。

## 9. 第三阶段目标

第三阶段做自动化和智能能力。

包括：

- SSE 实时推送
- AI 自动总结
- AI 今日建议
- 飞书提醒
- OpenClaw 联动
- Webhook
- MCP 查询和创建能力

## 10. 建议执行顺序

1. 跑通现有前后端。
2. 替换系统名称、Logo、描述。
3. 改造首页为 Life Manager 静态页面。
4. 新增项目管理静态页面。
5. 新增项目详情和项目管理静态页面。
6. 调整菜单。
7. 新增后端业务模块。
8. 页面接入真实接口。
9. 接入 OSS、AI、SSE。
10. 做 Docker 部署收口。

## 11. 当前风险

1. 前端不是 React，但 Vue 底座可以接受。
2. RuoYi-Vue-Plus 功能较重，需要隐藏不必要入口。
3. 当前后端默认 Java 版本配置为 17，如果坚持 JDK 21，需要后续验证构建兼容性。
4. 第一版如果同时追求完整后端、完整 UI、AI、OSS、MCP，会范围过大。

## 12. 当前建议

先做“静态产品壳”，让系统看起来就是 Life Manager。

也就是先完成：

- 首页
- 项目管理
- 无限暖暖项目页
- 日本旅行项目页

这些页面用假数据即可。等产品形态确认后，再开始写业务接口和数据库。
