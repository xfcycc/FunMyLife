# Life Manager 项目化需求设计文档

## 1. 文档范围

本文基于 FunMyLife / Life Manager 当前已经完成的六个页面整理：

| 页面 | 说明 |
| --- | --- |
| `/life/home` | Life Manager 首页，当前产品入口与每日概览页 |
| `/life/projects` | 项目列表与项目管理总览页 |
| `/life/project/infinity-nikki` | “无限暖暖”游戏项目详情页 |
| `/life/project/infinity-nikki/manage` | “无限暖暖”游戏项目管理配置页 |
| `/life/project/japan-travel` | “日本旅行 2026”旅行项目详情页 |
| `/life/project/japan-travel/manage` | “日本旅行 2026”旅行项目管理配置页 |

本文只描述产品需求、页面设计、交互规则、状态变化和信息组织方式，不包含技术实现、代码结构、接口字段或数据库设计。

## 2. 产品定位

Life Manager 是以“项目”为核心载体的个人生活管理系统。它不是单一待办工具，也不是传统后台表格系统，而是把生活中分散的日程、倒计时、清单、资产、图册、笔记、AI 建议和外部消息统一组织到项目空间中。

当前六个页面已经表达出两个核心项目类型：

| 项目类型 | 已有示例 | 项目承载内容 |
| --- | --- | --- |
| 游戏养成项目 | 无限暖暖 | 日常任务、周常任务、活动倒计时、版本直播、账号资产、素材分区、攻略笔记、截图图册、AI 项目建议 |
| 旅行计划项目 | 日本旅行 2026 | 出发倒计时、城市行程、行前清单、机票酒店、预算、天气地点、证件资料、同行人员、旅行图册、AI 行前建议 |

项目类型可以继续扩展到学习计划、家庭保险、健康管理、长期收纳、摄影灵感等生活主题，但扩展方式必须沿用当前页面已经建立的项目模型：一个项目聚合多个能力模块，首页和列表页负责跨项目总览，详情页负责项目日常使用，管理页负责项目规则配置。

## 3. 项目作为核心载体

### 3.1 项目的定义

项目是一个生活主题的长期容器。它至少包含：

| 组成 | 需求说明 |
| --- | --- |
| 基本信息 | 项目名称、类型、状态、封面、标签、简介、开始时间、创建时间、目的地或主题描述 |
| 任务与清单 | 根据项目类型承载今日任务、周常任务、行前清单、行李清单、预订清单等 |
| 时间节点 | 承载日程、活动结束时间、出发时间、版本直播时间、证件到期时间等 |
| 资产资料 | 承载账号、订单、票券、酒店、证件、保险、支付凭证、附件等 |
| 内容记录 | 承载笔记、攻略、旅行记录、游戏复盘、项目动态 |
| 图册素材 | 承载截图、照片、视频、活动图、旅行图集，并支持按项目归档 |
| AI 能力 | 根据项目中的任务、时间、资产、图册和记录生成建议、复盘、总结和风险提醒 |
| 外部频道 | 承接飞书、群聊、机器人、Webhook 等外部消息，并转为项目内结构化内容 |

### 3.2 页面关系

| 页面层级 | 页面 | 角色 |
| --- | --- | --- |
| 首页 | `/life/home` | 面向每天打开系统的第一屏，展示今日、本周、到期、项目、AI、快捷入口和实时状态 |
| 项目列表 | `/life/projects` | 面向跨项目管理，展示项目统计、筛选、搜索、排序、卡片/列表视图、模板和最近更新 |
| 项目详情 | `/life/project/{project}` | 面向项目日常使用，展示该项目内的任务、倒计时、资产、笔记、图册、AI 和最近动态 |
| 项目管理 | `/life/project/{project}/manage` | 面向项目初始化和规则配置，管理基础信息、模板、提醒、资产、图册同步、AI、外部通知 |

用户应能从首页进入项目，从项目列表进入项目详情或管理，从项目详情进入项目管理，从管理页保存配置后回到项目详情查看变化。

### 3.3 项目状态

项目至少需要表达以下状态：

| 状态 | 用途 | 当前页面体现 |
| --- | --- | --- |
| 进行中 | 项目活跃展示，可出现在首页和项目列表 | 无限暖暖、日本旅行 2026 |
| 已停用 | 项目规则暂停，任务、提醒、同步等能力不再主动触发 | 两个管理页均有启用/停用入口 |
| 已归档 | 项目不再日常关注，但保留历史资料 | 项目列表支持归档/恢复与归档筛选 |
| 需关注 | AI 未更新、任务偏慢、预算超限、提醒待处理等 | 项目列表“需关注”筛选，旅行预算状态，游戏周常状态 |
| 同步中/生成中 | 图册同步、AI 总结等异步能力正在处理 | 项目列表 AI 生成中，管理页图册同步中 |

## 4. 能力融合设计

Life Manager 的能力不是独立堆叠，而是围绕项目互相联动。

### 4.1 日程

日程用于描述具体时间安排。

| 场景 | 项目内角色 | 用户操作 | 联动 |
| --- | --- | --- | --- |
| 旅行行程 | 按城市、日期和时间展示景点、交通、餐厅、入住等安排 | 切换城市筛选、查看详情、打卡完成、上下滚动 | 影响旅行进度、AI 行前建议、最近动态、提醒规则 |
| 游戏版本节点 | 表达直播、维护、活动开放等时间节点 | 设置提醒、取消提醒、打卡节点、查看时间线 | 生成倒计时、进入提醒列表、影响 AI 项目分析 |
| 首页概览 | 汇总今日或本周日程数量 | 切换“今日/本周” | 更新首页今日概览数字 |

### 4.2 倒计时与提醒

倒计时用于突出临近时间节点，提醒用于主动通知用户。

| 能力 | 当前表达 | 需求细节 |
| --- | --- | --- |
| 活动倒计时 | 无限暖暖活动剩余 3 天、13 天、21 天 | 每个倒计时项显示标题、结束时间、剩余时间、提醒状态、打卡状态 |
| 出发倒计时 | 日本旅行显示 48 天 08 时 32 分 16 秒 | 倒计时应在项目详情头部高优展示，说明出发日期和星期 |
| 到期提醒 | 首页展示 30 天内即将到期项目 | 到期列表应显示资产名称、到期日期、剩余天数和类型图标 |
| 提醒规则 | 两个管理页均可新增、编辑、删除、启停提醒规则 | 支持提前时间、通知渠道、批量启用/停用、选中状态、停用状态 |

### 4.3 Todo 与清单

Todo 用于每日具体执行，清单用于项目准备和模板化复用。

| 类型 | 当前表达 | 需求细节 |
| --- | --- | --- |
| 首页今日待办 | 工作、生活、学习待办，可完成/恢复，可新增 | 待办显示标题、分类、时间、完成状态；新增时校验标题不能为空 |
| 游戏日常任务 | 今日、周常、活动三组任务 | 支持分组切换、完成/恢复、进度环、任务滚动 |
| 游戏周常目标 | 每个目标有当前值、目标值、状态 | 支持选中目标、加减进度、状态在推进中/需关注/已完成间变化 |
| 旅行行前清单 | 行前、行李、预订、其他分组 | 支持分组切换、完成/恢复、总进度、分组完成数 |
| 项目模板清单 | 管理页配置任务模板或清单模板 | 支持新增、编辑、删除、启用、停用、排序、批量操作 |

### 4.4 AI

AI 在项目内扮演“总结、建议、风险检查和复盘”的角色，不替代用户操作。

| 场景 | AI 输出 | 用户操作 | 边界 |
| --- | --- | --- | --- |
| 首页 AI 今日建议 | 基于日记、项目待办、活动提醒、学习计划给出今日建议 | 换一条、查看完整建议 | 只给建议，不自动修改用户数据 |
| 游戏 AI 项目建议 | 根据任务完成度、活动倒计时、素材收集和搭配目标生成建议 | 刷新建议、查看完整分析报告 | 不自动打卡，不自动设置提醒 |
| 旅行 AI 行前建议 | 根据行程、清单、预算、天气生成准备建议 | 重新生成、查看全部建议 | 涉及预算、证件、航班调整时必须用户确认 |
| 管理页 AI 设置 | 活动总结、日常复盘、攻略整理、版本解读、旅行总结、风险检查等 | 启用/停用 AI 规则，查看历史或管理规则 | AI 自动任务需明确启用状态和适用项目 |

### 4.5 资产

资产是项目中需要保管、追踪和提醒的重要资料。

| 项目类型 | 资产示例 | 需求细节 |
| --- | --- | --- |
| 游戏项目 | 官方账号、小号、充值账号、支付凭证、兑换码 | 默认只展示“已保护”“已绑定”等安全状态，不在概览暴露敏感内容 |
| 旅行项目 | 机票、酒店、JR Pass、护照、签证、保险、接送机、租车订单 | 资产显示名称、说明、状态，可进入详情，支持上传资料、启停和批量启停 |
| 首页到期 | 会员、域名、保险等生活资产 | 用“即将到期”卡片跨项目提醒 |

### 4.6 图床 / 相册 / OSS

图册能力用于把截图、照片、视频和素材按项目归档。

| 能力 | 当前表达 | 需求细节 |
| --- | --- | --- |
| 项目图册预览 | 游戏截图 8 个图册 126 张照片；旅行 128 张照片 12 个视频 | 支持横向滑动、点击照片查看详情、关联拍摄地点或笔记 |
| 图册同步 | 管理页显示 OSS 已连接、最近同步、同步内容、立即同步 | 支持开启/停用自动同步、立即同步、同步中状态、同步完成提示 |
| 素材分区 | 游戏项目有服装材料、活动兑换、拍照地点 | 分区支持新增、编辑、删除、启停、选中和滚动 |
| 旅行素材 | 旅行项目图册显示容量和同步数量 | 支持同步设置，未来可按城市、日期、同行人员、地点分类 |

### 4.7 最近动态

最近动态是项目内能力联动后的统一时间线。

| 来源 | 可形成动态 |
| --- | --- |
| 待办/清单 | 完成每日任务、行李清单更新 |
| 资产 | 机票出票、酒店预订成功、保单上传 |
| 图册 | 上传截图、上传旅行照片 |
| AI | AI 生成学习周报、项目总结完成 |
| 外部消息 | 群聊确认事项、机器人提醒、飞书消息转项目 |

动态支持横向或纵向滚动，点击动态可打开详情弹框。

## 5. 项目配置模板设计

项目模板用于把一类生活主题的默认结构复用到新项目中。当前项目列表页已经展示“游戏项目模板、旅行项目模板、学习项目模板、生活项目模板”，两个管理页进一步展示了模板内部配置项。

### 5.1 模板基础信息

| 配置项 | 说明 |
| --- | --- |
| 模板名称 | 如游戏项目模板、旅行项目模板 |
| 模板类型 | 游戏、旅行、学习、生活或自定义 |
| 模板说明 | 说明该模板适用场景 |
| 默认封面与图标 | 用于项目列表和详情页识别 |
| 默认标签 | 如游戏的换装、日常、活动、收集；旅行的自由行、亲友同行、摄影旅行 |
| 默认状态 | 新建项目后默认为进行中 |
| 是否启用 | 停用后不出现在新建项目推荐中 |

### 5.2 分区配置

模板应支持预设项目内分区。

| 分区类型 | 游戏模板 | 旅行模板 |
| --- | --- | --- |
| 详情页页签 | 概览、日常、倒计时、资产、笔记、图册、AI | 概览、行程、清单、资产、记录、图册、AI |
| 管理页页签 | 基础信息、日常模板、提醒规则、资产关联、图册同步、AI 设置 | 基础信息、行程规划、同行人员、清单模板、资产资料、图册同步、AI 设置 |
| 配置分区 | 基础配置、日常任务、素材分区、提醒规则 | 日期配置、行程模板、预算分类、证件资料 |

分区应支持启用、停用和排序。停用分区后，详情页不应主动展示该分区入口，但历史内容不删除。

### 5.3 任务模板

| 配置项 | 说明 |
| --- | --- |
| 名称 | 如每日基础任务、采集与材料收集、挑战关卡、周常任务 |
| 说明 | 描述任务模板用途 |
| 周期 | 每日、每周或项目自定义周期 |
| 重置时间 | 如每日 04:00、每周一 04:00 |
| 启用状态 | 启用后自动进入项目任务体系 |
| 选中状态 | 用于批量启用、批量停用、编辑和排序 |
| 排序 | 决定详情页任务显示顺序 |

### 5.4 提醒规则

| 配置项 | 说明 |
| --- | --- |
| 规则名称 | 如活动结束提醒、直播开始提醒、版本维护提醒、行程提醒、航班提醒 |
| 触发对象 | 活动、直播、维护、航班、酒店、证件、清单等 |
| 提前时间 | 如提前 30 分钟、提前 1 天、提前 7 天、提前 60 天 |
| 通知渠道 | 如飞书、邮件、浏览器通知、机器人、Webhook |
| 启用状态 | 停用后不触发 |
| 批量操作 | 选中后批量启用或停用 |

### 5.5 素材 / 资产分组

| 配置项 | 说明 |
| --- | --- |
| 分组名称 | 如服装材料、活动兑换、拍照地点、证件资料、机票交通 |
| 分组说明 | 描述归档对象 |
| 数量或金额 | 素材数量、清单数量、预算金额等 |
| 状态 | 已上传、待上传、待确认、已绑定、已停用 |
| 启用状态 | 决定是否进入项目详情页统计和提醒 |
| 批量操作 | 资料与清单支持批量启用/停用 |

### 5.6 预算 / 清单 / 证件

旅行模板需要包含预算、清单和证件配置。

| 配置项 | 需求 |
| --- | --- |
| 预算分类 | 机票交通、住宿酒店、餐饮购物、门票体验等，可新增、编辑、删除、启停 |
| 预算金额 | 每个分类有金额，启用分类参与总预算 |
| 清单模板 | 行前准备、行李打包、每日必备、应急药品等，显示条目数量 |
| 证件资料 | 护照、机票行程单、酒店预订单、旅行保险单、签证资料等，显示上传或确认状态 |
| 同行人员 | 记录姓名、职责、角色、启用状态 |

### 5.7 AI 规则

| 项目类型 | AI 规则 |
| --- | --- |
| 游戏 | 活动总结、日常复盘、攻略笔记整理、版本更新解读 |
| 旅行 | 旅行总结、行前建议、行程风险检查、每日回顾 |

AI 规则必须支持独立启用/停用。系统需要显示 AI 任务历史或完整规则管理入口。AI 自动化结果应进入项目动态，并可在详情页 AI 卡片查看。

### 5.8 同步设置

| 配置项 | 说明 |
| --- | --- |
| 同步开关 | 控制项目图册或素材是否自动同步 |
| 同步状态 | 正常、暂停、同步中、同步成功、同步失败 |
| 最近同步时间 | 告知用户上次同步结果 |
| 同步内容 | 截图、活动图、视频、旅行照片等 |
| 立即同步 | 用户手动触发同步 |
| 同步规则 | 管理项目内素材分类、路径和同步范围 |

### 5.9 模板复用、启停、排序、批量操作

| 操作 | 需求 |
| --- | --- |
| 复用模板 | 新建项目时选择模板，自动带出页签、任务、提醒、资产分组、AI 规则和同步设置 |
| 启用/停用模板 | 停用模板后，不再作为新建项目推荐；已有项目不受影响 |
| 启用/停用模板项 | 单个任务模板、提醒规则、资料、清单、素材分区均可启停 |
| 排序 | 任务模板和行程日可排序，排序结果影响详情页展示 |
| 批量操作 | 选中多个模板项后批量启用或停用 |
| 删除 | 删除前必须二次确认；删除配置项只移除配置，不应误删历史记录 |

## 6. 首页小组件设计

首页小组件用于实现个性化首页和模板化首页。当前首页已包含今日概览、今日待办、即将到期、项目总览、AI 今日建议、本月统计、快捷入口、实时状态；项目详情页也已经表现出可复用的小组件形态。

### 6.1 小组件通用规则

| 规则 | 说明 |
| --- | --- |
| 可扫读 | 标题短、数字突出、状态明确 |
| 可操作 | 每个小组件至少有一个进入详情、切换状态或执行动作的入口 |
| 可反馈 | 操作后通过提示反馈结果 |
| 可空态 | 没有数据时说明原因并提供下一步入口 |
| 可排序 | 首页小组件后续应支持拖动排序或配置排序 |
| 可隐藏 | 用户可以关闭不关注的小组件 |
| 可项目化 | 小组件既可展示全局数据，也可绑定单个项目 |

### 6.2 小组件清单

| 小组件 | 用途 | 状态 | 交互 | 空状态 |
| --- | --- | --- | --- | --- |
| 今日任务 | 展示今天要完成的待办或项目任务 | 未完成、已完成、已恢复 | 点击完成/恢复，添加待办，切换今日/本周 | “今天暂无待办”，提供“添加待办” |

## 8. 外部频道接入

外部频道用于把飞书、群聊、机器人或其他消息纳入项目体系。它们不能绕过用户确认直接修改关键数据。

### 8.1 接入对象

| 外部频道 | 角色 |
| --- | --- |
| 飞书机器人 | 接收提醒、推送项目动态、接收用户转发的消息 |
| 飞书群聊 | 承接多人旅行、协作计划、群内确认事项 |
| 私聊机器人 | 快速创建待办、补录动态、查询项目状态 |
| Webhook | 接收外部系统触发的文件、日程、支付或自动化事件 |
| OpenClaw / 自动化工具 | 承接自动化执行结果或任务状态 |

### 8.2 外部消息转入项目

| 外部消息类型 | 可转换为 | 需要用户确认 |
| --- | --- | --- |
| “明天 10 点去浅草寺” | 旅行项目日程、提醒、项目动态 | 确认项目、时间、地点、是否提醒 |
| “护照照片已上传” | 资产资料、证件资料、图册动态 | 确认资料类型、可见范围、是否关联旅行项目 |
| “今晚 20:30 学习打卡” | Todo、日程、提醒 | 确认是否重复、周期和提醒渠道 |
| “版本直播 04-30 19:30” | 游戏项目倒计时、提醒、动态 | 确认项目、时间、提前提醒规则 |
| “机票订单截图” | 旅行资产、图册、项目动态 | 确认证件/订单分类，避免误归档 |
| 群聊多条讨论 | AI 总结、待办候选、行程候选 | 确认摘要是否写入项目，确认待办责任人 |

### 8.3 边界与安全

1. 外部消息默认先进入“待确认”状态，不直接写入关键资产或日程。
2. 涉及证件、账号、支付、保险等敏感资料时，必须由用户确认归属项目和可见范围。
3. AI 可以提取候选待办、日程、资产、动态，但不能自动删除或覆盖已有内容。
4. 群聊内容进入个人项目时，需要提示来源和时间，避免误认为用户手动录入。
5. 外部频道生成的提醒应显示渠道来源，用户可在项目管理页启停规则。
6. 外部消息无法识别项目时，应进入收件箱或待分配区，而不是随机归档。

| 管理提醒规则 | 新增、编辑、删除、启停通知规则 |

| 新增预算分类 | 打开预算分类弹框 |
| 启停/编辑/删除预算分类 | 预算分类和总预算变化 |
| 预览项目概览 | 提示跳回日本旅行项目概览 |

## 10. MVP 边界

### 10.1 当前页面已经表达出来的 MVP 需求

| 需求 | 依据 |
| --- | --- |
| 以项目为核心组织生活内容 | 首页项目总览、项目列表、两个项目详情页和两个管理页 |
| 首页每日概览 | 今日/本周、待办、到期、项目、AI、统计、快捷入口、实时状态 |
| 项目列表管理 | 统计、筛选、搜索、排序、视图切换、新建、编辑、归档、模板、最近更新 |
| 游戏项目详情 | 日常、周常、活动倒计时、版本更新、账号资产、笔记、图册、AI、动态 |
| 游戏项目配置 | 基础信息、重置时间、任务模板、提醒规则、资产、素材、图册同步、AI、集成 |
| 旅行项目详情 | 出发倒计时、行程、清单、资产、预算、天气、图册、笔记、AI、动态 |
| 旅行项目配置 | 基础信息、行程日、同行人员、清单模板、证件资料、图册同步、通知、AI、预算 |
| 统一交互反馈 | 弹框、成功/信息/警告提示、启停、选中、排序、批量操作 |
| 小组件化首页 | 今日任务、倒计时、资产提醒、AI 建议、预算、图册、项目进度、实时状态等 |
| 外部集成入口 | 飞书机器人、OpenClaw、Webhook、通知渠道 |

### 10.2 MVP 应保留的边界

1. 第一阶段以现有六个页面的需求闭环为边界，不新增完整社交协作、复杂权限、财务记账系统或全量 AI Agent 自动执行。
2. AI 只做建议、总结、复盘、候选提取和风险提示，不自动替用户修改关键日程、资产、预算和证件资料。
3. 外部频道先做“待确认转入项目”，不做无确认的自动写入。
4. 资产展示先保护敏感信息，只显示状态、名称和安全入口。
5. 模板管理先覆盖游戏和旅行两类已经落地的模板，学习和生活模板只做列表入口和新建项目类型支持。
6. 首页个性化先支持筛选、置顶、切换和快捷入口，拖拽布局、复杂权限和跨设备布局同步可后置。
7. 项目管理先支持配置项的新增、编辑、删除、启停、排序和批量操作，不扩展到复杂审批流程。

## 11. 待确认问题

1. 首页实际产品路径是否统一为 `/life/home`，还是沿用当前入口 `/life` 并把 `/life/home` 作为别名。
2. 项目详情页的页签下一阶段是否需要真实切换内容，还是继续以概览页聚合全部模块。
3. 外部频道第一阶段优先接入飞书机器人、飞书群聊、Webhook 中的哪一个。
4. 敏感资产的展示边界需要进一步确认：是否需要二次验证后才允许查看账号、证件和订单详情。
5. 项目模板是否允许用户复制已有项目生成模板，还是只允许从模板中心新建模板。
