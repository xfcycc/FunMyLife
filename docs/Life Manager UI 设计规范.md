# Life Manager UI 设计规范

本文档基于当前已实现的首页、项目管理、无限暖暖项目页、日本旅行项目页及两个项目管理页整理，用于后续交互演示和通用组件封装。

## 1. 设计定位

Life Manager 是个人生活管理产品，不走传统后台管理系统的强表格风格。界面应保持轻量、柔和、信息密集但不拥挤，重点服务“每天快速知道该关注什么”。

设计关键词：

- 清爽：大面积浅色背景，避免沉重后台感。
- 柔和：紫色作为品牌主色，配合粉、蓝、绿、琥珀色表达生活场景。
- 克制：圆角、阴影、渐变都只用于建立层级，不做装饰性堆叠。
- 可扫读：卡片标题短、数字突出、状态标签明确。

## 3. 字体与排版

### 3.1 字体栈

页面使用：

```css
Inter, 'PingFang SC', 'Microsoft YaHei', Arial, sans-serif
```

要求：

- 不使用负字距，统一 `letter-spacing: 0`。
- 中文文本保持自然行高，不压缩阅读空间。

### 3.2 字号层级

| 层级 | 字号 | 行高 / 字重 | 使用场景 |
| --- | --- | --- | --- |
| 页面主标题 | `21px` | `1.18` / `850` | 顶部 `h1` |
| 项目详情标题 | `24px` | `1.18` / `850` | 项目详情头部 |
| 大型封面标题 | `29px` | 约 `1.15` / `800` | 旅行封面、视觉 Hero |
| 卡片标题 | `13px` | `800` | `lm-card-title h2` |
| 正文 | `12px` | 常规 | 页面说明、卡片主体 |
| 辅助文本 | `10px` - `11px` | 常规 | 标签、时间、备注 |
| 数据数字 | `16px` - `28px` | `800` | 统计、进度、倒计时 |

## 4. 间距规范

### 4.1 页面级

- 侧边栏宽度：桌面 `168px`，中等屏 `150px`。
- 主内容内边距：桌面 `20px 24px 18px`，移动端 `16px`。
- 顶栏底部间距：`18px`。
- 页面网格间距：`14px` 为默认业务卡片间距，旧版 Life 卡片可用 `16px`。

### 4.2 卡片级

- 默认卡片内边距：`15px`。
- 管理类信息卡可按内容使用 `13px` - `20px`。
- 卡片标题与内容间距：`14px` - `16px`。
- 列表项间距：`10px` - `12px`。
- 图标与文字间距：`7px` - `11px`。

## 5. 圆角与阴影

圆角遵循“克制、后台工具友好”的原则。

| 元素 | 圆角 |
| --- | --- |
| 主卡片 / 面板 / Hero | `8px` |
| 导航项 / 小按钮 / 标签 | `6px` |
| 小封面 | `7px` - `8px` |
| 胶囊标签 / 进度条 / 开关 | `999px` |
| 头像 | `50%` |

阴影：

- 卡片：`0 12px 28px rgba(58, 66, 100, 0.07)`。
- 主按钮：`0 10px 20px rgba(111, 83, 231, 0.28)`。
- 侧边栏：`10px 0 28px rgba(46, 54, 92, 0.03)`。
- 浮层：使用更高层级阴影，但透明度保持低，避免厚重。

## 6. 组件规范

### 6.1 按钮

默认按钮高度 `34px`，圆角 `8px`。

- 主按钮：紫色渐变，白色文字，用于“新建、保存、确认”。
- 次按钮：白底浅阴影，用于“取消、分享、设置、查看”。
- 图标按钮：`34px * 34px`，用于搜索、通知、更多等高频工具。
- 圆形新增按钮：`36px` - `38px`，仅用于顶栏快捷创建。

### 6.2 卡片

卡片是主要信息容器：

- 使用浅边框、白色半透明背景和轻阴影。
- 标题区左侧可以带 `22px` 小图标底。
- 卡片内部避免再嵌套完整卡片，只使用浅底行、分割线或列表。

### 6.3 标签

默认标签高度 `18px`，圆角 `6px`，字号 `10px`。

标签只表达类型和状态，不承载长句。建议类型：

- 紫色：默认 / AI / 通用。
- 绿色：进行中 / 已完成 / 已连接。
- 蓝色：旅行 / 信息。
- 粉色：游戏 / 温柔提醒。
- 琥珀色：活动 / 到期 / 重要。

### 6.4 弹框

弹框用于创建、编辑、确认、详情预览：

- 弹框宽度默认 `420px`，表单类可扩展到 `560px`。
- 圆角 `8px`，内边距 `18px`。
- 标题字号 `15px`，说明文字 `12px`。
- 底部按钮右对齐，取消在左，确认在右。
- 遮罩使用浅冷色透明层，不做深黑遮罩。

### 6.5 提示框

提示框用于交互反馈：

- 宽度建议 `320px` - `360px`。
- 位置默认右上角。
- 圆角 `8px`，浅边框，轻阴影。
- 成功、警告、错误、信息分别使用绿色、琥珀、红色、紫色图标底。
- 文案短句优先，标题不超过 12 个中文字符。

## 7. 响应式规则

- `1180px` 以下，四列及以上网格降为两列。
- `820px` 以下，侧边栏变为顶部区域，导航只保留图标。
- 移动端主内容内边距 `16px`。
- 按钮组允许换行，但按钮文本不能被压缩到溢出。
- 固定格式元素需要有稳定宽高或网格轨道，避免 hover 和动态数据引起布局跳动。

## 8. 后续页面落地约束

1. 新页面优先复用 `LifeAppShell`、`lm-card`、`lm-tabs`、`lm-tag`、`lm-purple-btn`、`lm-plain-btn`。
2. 交互演示页新增弹框、提示、确认反馈时，优先使用 `LifeModal`、`LifeToastHost` 与 `useLifeToast`。
3. 不新增明显不同的圆角、阴影和主色。
4. 不使用大面积单一紫色背景；紫色只做品牌和关键操作强调。
5. 页面第一屏必须直接展示业务内容，不做营销式空 Hero。
