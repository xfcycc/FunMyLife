# P.A.C.E. 复杂度路由 v9.2.0

收到任务后, 按以下规则选择路径:

| 条件 | 路径 | 阶段 | Codex 参与 |
|------|------|------|-----------|
| 修 bug / 改配置 / <30min | **A (快速)** | E→T | 可选 |
| 新功能 / 重构 / 30min-4h | **B (标准)** | R₀→R→D→P→E→T→V | P对抗审查 + E委托 + T互审 |
| 跨模块 / 架构变更 / >4h | **C (复杂)** | 同B + 子代理并行 | 同B + 多任务并行 |
| 多服务 / 全栈 / 多人协作 | **D (系统)** | 同C + 设计评审 | 同C |

选定路径后 → 读 riper-7.md 执行对应阶段。
Path A 跳过 R₀/R/D/P, 直接 E→T。
